Cómo piensa
un programa

Antes de aprender un lenguaje, hay que aprender la lógica que hay debajo de todos ellos. Este es el mapa: las piezas con las que se construye cualquier código, explicadas sin tecnicismos.

Formatopseudocódigo genérico Duración2–3 horas Nivelinicial
Antes de empezar — el principio detrás de todo

Casi todo se reduce a verdadero o falso

Da igual el lenguaje — Python, JavaScript, Java, C++...— por debajo, casi toda la lógica de un programa se apoya en preguntas que solo admiten dos respuestas: VERDADERO o FALSO. "¿Es mayor de 18?", "¿Coincide la contraseña?", "¿Quedan elementos en la lista?". Cada condicional, cada bucle y muchas funciones existen para responder una de estas preguntas booleanas y decidir qué instrucción se ejecuta a continuación.

● VERDADERO · "true" ● FALSO · "false"
Módulo 00

¿Qué es programar, en realidad?

La analogía de la receta. Programar es escribir instrucciones tan precisas que alguien sin criterio propio —el ordenador— pueda seguirlas sin equivocarse. Igual que una receta dice "bate 2 huevos, añade 200g de harina, hornea 20 minutos a 180°", un programa dice "guarda este número, compáralo con este otro, repite esto 5 veces".
Las piezas que veremos
  • Variables — dónde se guardan los datos ("variable")
  • Tipos de dato — qué clase de información es ("data type")
  • Operadores — cómo se combina y compara ("operators")
  • Condicionales — cómo se toman decisiones ("if / else")
  • Bucles — cómo se repite algo ("loop")
  • Funciones booleanas — cómo se responde sí/no ("boolean function")
  • Funciones — cómo se reutiliza lógica ("function")
  • Listas — cómo se agrupan varios datos ("array")
  • Seguridad — cómo se cuela código que no pediste ("dependency")
La notación que usaremos
  • No usaremos un lenguaje real todavía
  • Escribiremos en pseudocódigo: instrucciones en español, con una estructura fija
  • Palabras clave como SI, MIENTRAS, FUNCION marcan cada pieza
Módulo 01

Variables: cajas con nombre "variable"

La analogía de la caja etiquetada. Una variable es una caja donde guardas un dato, con una etiqueta pegada para poder encontrarla luego. No importa lo que haya dentro hoy — mañana puedes vaciarla y meter otra cosa distinta. Lo único fijo es la etiqueta: el nombre.
En una fraseUna variable es un espacio con nombre que guarda un dato que puede cambiar mientras el programa se ejecuta. En la mayoría de lenguajes se declara con let, var o const, y se asigna con el signo = ("assign").
GUARDAR "María" EN nombre   // "store" ... "in"
GUARDAR 34 EN edad
GUARDAR edad + 1 EN edad   // la caja "edad" ahora vale 35
Ejercicio en grupo — 5 min

En papel: escribe 3 variables que necesitarías para representar a un cliente de una tienda. Luego escribe una línea de pseudocódigo que actualice una de ellas.

Pista Piensa en datos que cambian con el tiempo (edad, puntos de fidelidad, saldo) frente a datos que no cambian (DNI).
Módulo 02

Tipos de dato: qué clase de cosa es "data type"

No todas las cajas guardan lo mismo. Un programa necesita saber si lo que hay dentro es un número, una palabra o una respuesta de sí/no, porque cada tipo permite operaciones distintas.

TipoNombre en inglésEjemplo
Texto"string""Antonella"
Entero"integer" / "int"42
Decimal"float" / "double"19.90
Booleano"boolean" / "bool"true / false
Lista"array" / "list"["rojo", "azul"]
En una fraseEl tipo de dato es la naturaleza de lo que hay en la caja, y determina qué operaciones tienen sentido sobre ella.
Módulo 03

Operadores: verbos del código "operators"

Aritméticos · "arithmetic"
  • + suma — "add"
  • - resta — "subtract"
  • * multiplicación — "multiply"
  • / división — "divide"
Comparación · "comparison"
  • == es igual a — "equal"
  • != es distinto de — "not equal"
  • > / < mayor / menor que — "greater" / "less than"

Los operadores de comparación no devuelven un número: devuelven un booleano ("boolean"), VERDADERO ("true") o FALSO ("false").

GUARDAR edad > 18 EN esMayorDeEdad   // = VERDADERO ("true") o FALSO ("false")
Módulo 04

Condicionales: el programa decide "if / else"

La analogía del cruce de caminos. Un condicional ("conditional") es una bifurcación: "si esto es verdad, ve por aquí; si no, ve por allá".
En una fraseUn condicional ejecuta un bloque u otro dependiendo de si una condición es VERDADERA o FALSA.
SI edad > 18 ENTONCES   // "if" ... "then"
    MOSTRAR "Puedes entrar"
SINO   // "else"
    MOSTRAR "Eres menor de edad"
FIN SI
Ejercicio en grupo — 10 min

Escribe la lógica de una máquina expendedora: si el dinero introducido es suficiente, entrega el producto; si no, muestra cuánto falta.

Pista Dos variables (precio, dineroIntroducido) y un SI/SINO.
Módulo 05

Bucles: repetir sin copiar y pegar "loop"

La analogía de la cinta de correr. Un bucle repite un tramo de instrucciones mientras se cumpla una condición, en vez de escribir la misma línea 50 veces.
MIENTRAS — "while"
GUARDAR 0 EN contador
MIENTRAS contador < 5
    MOSTRAR contador
    GUARDAR contador + 1 EN contador
FIN MIENTRAS
PARA CADA — "for" / "for each"
PARA CADA numero EN [1,2,3,4,5]
    MOSTRAR numero * 2
FIN PARA

↳ Este bloque se retoma con más profundidad en el siguiente curso.

Módulo 06

Funciones booleanas: máquinas de sí o no "boolean function"

En una fraseUna función booleana evalúa una pregunta sobre uno o varios datos y solo puede devolver VERDADERO ("true") o FALSO ("false").
Y "and"
Se cumplen los dos términos
A && B → true
O "or"
Se cumple al menos uno
A || B → true
NO "not"
Se cumple A, excluyendo B
A && !B → true
FUNCION puedeEntrar(edad, tieneEntrada)   // "function"
    DEVOLVER edad > 18 Y tieneEntrada   // "return"
FIN FUNCION
Módulo 07

Funciones: bloques reutilizables "function"

La analogía de la máquina de café. Una función recibe parámetros ("parameters") y devuelve ("return") un resultado, sin que tengas que saber cómo funciona por dentro cada vez. Las funciones booleanas son solo un caso particular: su único resultado posible es VERDADERO o FALSO.
FUNCION calcularDescuento(precio, porcentaje)
    GUARDAR precio * (porcentaje / 100) EN descuento
    DEVOLVER precio - descuento
FIN FUNCION

GUARDAR calcularDescuento(50, 20) EN precioFinal   // = 40
Ejercicio en grupo — 10 min

Escribe puedeComprar(edad, precio, dinero) usando internamente la lógica booleana del módulo anterior.

Pista DEVOLVER edad > 18 Y dinero >= precio.
Módulo 08

Listas: varias cajas en fila "array" / "list"

La analogía del casillero. Una lista es un mueble con casillas numeradas. En vez de crear una variable por cliente, se guardan todos en una lista y se recorre con un bucle.
GUARDAR ["María", "Luca", "Sofía"] EN clientes

PARA CADA cliente EN clientes
    MOSTRAR "Hola, " + cliente   // "show" / "print"
FIN PARA
Módulo 09

Seguridad: cuando el código no es tuyo "dependency"

La analogía del proveedor de confianza. Casi ningún programa se escribe desde cero: se construye importando "dependencias" — código publicado por otras personas en repositorios como npm, PyPI o GitHub. Al instalar una dependencia, tu máquina no solo copia ese código: a menudo lo ejecuta automáticamente. Si confías en el proveedor equivocado, le estás dando las llaves de tu ordenador.
En una fraseUna dependencia comprometida puede ejecutar código no deseado en tu máquina en el momento mismo de instalarla, no solo cuando la usas.
Formas habituales de colar código
  • Typosquatting — un paquete con nombre casi idéntico a uno popular (reqeusts en vez de requests)
  • Dependency confusion — un paquete "interno" con el mismo nombre se publica en el repositorio público con una versión más alta
  • Cuenta de mantenedor robada — alguien publica una versión nueva de una librería de confianza con código añadido
  • Constantes ofuscadas — un dato "inocente" (una cadena codificada) que en realidad se decodifica y ejecuta al instalar
Buenas prácticas del grupo
  • Fijar versiones exactas con un lockfile, nunca instalar "la última" a ciegas
  • Revisar quién mantiene la librería y desde cuándo, antes de añadirla
  • No copiar y pegar comandos de instalación sin entender qué hacen
  • Usar herramientas de auditoría (npm audit, pip-audit) antes de desplegar
GUARDAR "aHR0cDovL2Rlc2Nvbm9jaWRvLmV4YW1wbGU=" EN configUrl   // parece un dato cualquiera...

SI instalando ENTONCES
    EJECUTAR decodificar(configUrl)   // ...pero se decodifica y ejecuta solo, sin pedir permiso
FIN SI

Fíjate en la condición: SI instalando es otra vez una función booleana (módulo 06) — solo que aquí decide algo peligroso. La lógica que aprendisteis hoy sirve tanto para construir como para reconocer cuándo alguien la usa en tu contra.

Módulo 10 — Cierre

Las ocho piezas, juntas

Este programa calcula si cada cliente de una lista tiene descuento por ser mayor de 65 años. Usa variables, tipos de dato, un operador de comparación, una función, un condicional y un bucle recorriendo una lista. Todo lo visto hoy, combinado.

FUNCION calcularPrecio(edad, precioBase)
    SI edad > 65 ENTONCES
        DEVOLVER precioBase * 0.8
    SINO
        DEVOLVER precioBase
    FIN SI
FIN FUNCION

GUARDAR [25, 70, 40, 68] EN edadesClientes

PARA CADA edad EN edadesClientes
    GUARDAR calcularPrecio(edad, 20) EN precioFinal
    MOSTRAR "Cliente de " + edad + " años paga " + precioFinal
FIN PARA

Glosario rápido

Variable · "variable"Espacio con nombre que guarda un dato que puede cambiar.
Tipo de dato · "data type"Naturaleza del contenido: texto, número, booleano, lista.
Operador · "operator"Símbolo que combina o compara valores.
Booleano · "boolean"Valor que solo puede ser VERDADERO ("true") o FALSO ("false").
Condicional · "if statement"Bifurcación que ejecuta un bloque u otro según una condición.
Bucle · "loop"Bloque que se repite mientras se cumpla una condición.
Función booleana · "boolean function"Función cuyo único resultado posible es "true" o "false".
Función · "function"Bloque con nombre, reutilizable, que puede devolver ("return") un resultado.
Parámetro · "parameter"Dato que se le entrega a una función al llamarla.
Lista · "array" / "list"Colección ordenada de varios datos bajo una sola variable.
Dependencia · "dependency"Código de otra persona que tu programa importa y, a menudo, ejecuta automáticamente al instalarlo.
